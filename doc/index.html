Up 
post /appointment
adds an appointment (addAppointment)
Adds an appointment to the system

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body AppointmentItem (optional)
Body Parameter — Appointment to add

Responses

201

appointed created

400

invalid input, object invalid

409

an existing appointment already exists for the date
Up 
get /appointment
searches scheduled appointments (listAppointments)
By passing the userId, you can search for all the appointments available for the user

Query parameters

userId (required)
Query Parameter — pass an User Id for retrieve all the appointments availables format: email

Return type

array[AppointmentItem]

Example data

Content-Type: application/json
[ {
  "userId" : "mmarrari@sparkdigital.com",
  "startDate" : "2000-01-23 04:30:00"
}, {
  "userId" : "mmarrari@sparkdigital.com",
  "startDate" : "2000-01-24 15:00:00"
} ]

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

search results matching criteria

400

bad input parameter
